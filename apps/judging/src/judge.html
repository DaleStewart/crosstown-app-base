01 · The app

What is Crosstown?

A reference voice + text agentic app for an MTA dispatcher persona. Two specialists, six tools between them, every answer grounded in mock log corpora with explicit citations. Built so teams can fork it and replace one piece to match their hackathon use case.

Why coaches care What it means
Hour-1 baseline ships working A team deploys in < 20 min with azd up, then spends the hackathon extending — not chasing "Hello World".
Citations are load-bearing Every tool response carries citations[]. CI fails if > 5% of turns are uncited. This is the headline reliability story.
Voice provider is pluggable Foundry Realtime primary, Azure Speech fallback. Same orchestrator code; env-var swap.
Identity is keyless One user-assigned managed identity across all containers; DefaultAzureCredential everywhere. No API keys in env files.

07 · Quick start

From clone to talking · 5 steps

First-time deploy on a clean Azure subscription. ~15–20 minutes.

# Command Expected result
1 git clone <fork-url> && cd mta-ai-hackathon ~70 MB repo · all extensions + cassettes included
2 azd auth login Opens browser · pick the Devpost-provisioned subscription
3 azd up ~15–20 min · prints the frontend URL when done
4 scripts/smoke-test.sh <frontend-url> Should print ✓ smoke passed · fails fast if deploy didn't fully land
5 Open URL → hold mic / press space → speak "Show me the last hour of L2 events" → cited response within ~5s

If a team sees "Hello World"

  1. Confirm azure.yaml at repo root references all three services (not just frontend).
  2. Re-run azd deploy — provision succeeded, image push failed.
  3. If it persists, the ACA quickstart placeholder image is being served. scripts/smoke-test.sh catches this immediately. PR #34/#35 era pain.

12 · FAQ

Coach FAQ

The eight stuck points that come up most often.

Why does the frontend show "Hello World" instead of the chat UI?
ACA is serving the quickstart placeholder image — image push didn't land. Re-run azd deploy (not azd up). If it persists, check azure.yaml at repo root references the frontend service. scripts/smoke-test.sh catches this in < 5s.
Assistant responds but the user transcript never renders
Two recent fixes: (a) Foundry input-transcription not enabled — fixed in PR #42 + PR #38 (provisioned gpt-4o-mini-transcribe). (b) Frontend not rendering the user-transcript event — fixed in PR #37 + #43. If still missing, confirm orchestrator env has AZURE_OPENAI_TRANSCRIBE_DEPLOYMENT=gpt-4o-mini-transcribe.
Voice loop hangs after one turn
Pre-#20 era: orchestrator wasn't sending the audio-commit boundary, so Foundry never closed the input buffer. Server VAD + explicit commit landed in #20. Stop button + barge-in (#45) makes this user-visible. If a team's on a fork before #20, that's the merge to get.
Tool fires but returns 500 — they say their code is right
99% of the time it's the citation contract. Every tool handler must return a ToolResponse with non-empty citations[] OR explicitly emit warnings: ["uncited"]. The framework wraps handler exceptions into 200s with citations + warnings, but explicit 500s skip that. See apps/log_analyst/citations.py.
Should I let teams pin a stricter model than gpt-realtime-1.5?
No. The repo standardizes on gpt-realtime-1.5-2026-02-23 (Decision D-009). gpt-4o-realtime-preview is GA-deprecated. gpt-realtime-mini is the cost-optimized alternate if quota is tight. gpt-realtime-translate is for translation use cases only.
Team wants to skip citations to make their eval pass
Citation is the headline reliability story. If you let them dial --max-uncited-pct down, they've lost the demo. The right move: help them surface the citation that's actually in the tool result. ToolResponse.finalize() auto-tags warnings: ["uncited"] when empty — chase that warning back to the handler.
Team's not on App Modernization but wants to fork this
Yes, with caveats. Voice + grounding applies to most use cases. Steer them toward Ext 02 (swap corpus) and Ext 07 (rebrand). If their use case is Power Apps / Copilot Studio / Power Pages, this isn't their repo — flag a Track 1 coach.
What's the difference between this judging app and the Crosstown app?
Two separate Static Web Apps. This judging app (mtahack-swa-…) is for coaches — scorecards, admin, this runbook. The Crosstown app (frontend.blackriver-…) is what participants build on. Same identity provider (GitHub), different role gates.
