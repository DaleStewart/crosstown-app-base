# MTA AI Hackathon Accelerator Constitution

## Core Principles

### I. Citations Are Load-Bearing (NON-NEGOTIABLE)

**Ethos:** Every agent response earns trust through provenance — no unsourced claims reach the user.

Every Log Analyst tool returns a `ToolResponse` with `citations: list[Citation]` typed as `log | runbook | incident`. `ToolResponse.finalize()` auto-tags `warnings: ["uncited"]` when citations are empty. The citation eval gate (`evals/runner.py`) enforces ≤5% uncited turns on every PR; the orchestrator gate checks that user-facing replies contain citation tokens. Pinned IDs (`must_cite_ids` in scenarios) catch "cited *something*, just not the *right* something." Do not silently drop citations to make tests pass — fix the tool response instead.

**Enforced in:** `apps/log_analyst/citations.py`, `evals/runner.py`, `.github/workflows/eval.yml` (job `citation-gate`), `evals/scenarios/SC-*.yaml`.

### II. Mock Data Only (NON-NEGOTIABLE)

**Ethos:** Nothing in this repo references real MTA systems, employees, schedules, or telemetry — ever.

All files under `data/` are synthetic. Rail lines are fictional (L1/L2/L3). Pattern signatures in `detect_pattern` match synthetic event shapes generated by `data/generate_mock_data.py`. This protects the MTA's operational data, keeps the repo redistributable, and ensures eval/red-team gates exercise the architecture rather than production secrets. If your extension needs domain data, generate it.

**Enforced in:** `data/`, `data/generate_mock_data.py`, `README.md` ("Mock data only" guardrail), `.github/copilot-instructions.md`.

### III. Hermetic by Default, Live on Demand

**Ethos:** Every quality gate runs offline from cassettes; one env var flips to a deployed endpoint.

Eval and red-team runners replay `cassettes/<scenario-id>.json` for deterministic, zero-secret, sub-second CI. Setting `EVAL_MODE=live` plus a service URL env var (`LOG_ANALYST_URL`, `ORCHESTRATOR_URL`) switches to the deployed endpoint. This lets CI run without Azure credentials while still supporting integration smoke tests against a real subscription. When adding a scenario YAML, always add a matching cassette.

**Enforced in:** `evals/runner.py`, `evals/orchestrator_runner.py`, `redteam/runner.py`, `evals/cassettes/`, `.github/workflows/eval.yml`, `.github/workflows/redteam.yml`.

### IV. Keyless Auth Everywhere

**Ethos:** All Azure SDK clients authenticate via `DefaultAzureCredential` against a single user-assigned managed identity — no API keys, no secrets in env files.

A single UAMI is attached to all three Container Apps and granted data-plane roles in `infra/modules/roleAssignments.bicep`. Secrets that must exist (connection strings, App Insights keys) flow through Key Vault. The user running `azd up` gets equivalent dev-time roles for local debugging. Never add API-key auth or `AZURE_*_KEY` variables — the architecture assumes keyless.

**Enforced in:** `infra/modules/roleAssignments.bicep`, `infra/main.bicep` (UAMI attachment), `apps/orchestrator/settings.py`, `apps/log_analyst/settings.py`, `.github/copilot-instructions.md` (contract #5).

### V. One Voice Abstraction, Two Implementations

**Ethos:** `VOICE_PROVIDER` selects Foundry Realtime or Azure Speech Services; both implement `voice.base.VoiceProvider` — never bypass it.

The frontend is provider-agnostic — it streams audio over WebSocket and receives transcripts + audio back. Switching providers is a server-side env flip + redeploy. The orchestrator's `/api/turn` (text eval surface) and `/ws/voice` (live audio) share the same tool-routing and citation contract. Adding a third voice provider means implementing the `VoiceProvider` / `VoiceSession` interface — not forking the orchestrator.

**Enforced in:** `apps/orchestrator/voice/base.py`, `apps/orchestrator/voice/foundry_realtime.py`, `apps/orchestrator/voice/speech_services.py`, `apps/orchestrator/settings.py` (`VOICE_PROVIDER`), `.github/copilot-instructions.md` (contract #4).

### VI. Extensions Are Exercises, Not Features

**Ethos:** Each extension ships failing tests and acceptance criteria; teams make the tests pass — the skeleton never pre-implements them.

The nine `docs/extensions/NN_*/` folders are 30–60 min slices that forking teams pick based on their submitted use case. The skeleton stays minimal (the Hour-1 voice → orchestrator → specialist → cited-reply loop). This keeps the baseline deployable in under 20 minutes via `azd up` and ensures teams learn by doing rather than reading pre-built code.

**Enforced in:** `docs/extensions/01_*` through `docs/extensions/09_*`, `README.md` ("Extensions are exercises, not pre-built features"), `.github/copilot-instructions.md`.

## Quality Gates

Four CI gates enforce the contracts above on every PR to `main`:

| Gate | Workflow | Threshold | What it grades |
|---|---|---|---|
| **Citation** | `eval.yml` → `citation-gate` | ≤5% uncited turns | Tool-layer responses carry citations of the right type; pinned IDs where applicable |
| **Orchestrator** | `eval.yml` → `orchestrator-gate` | 0% scenario failures | User-visible reply contains required substrings + citation token; tool-routing correctness |
| **Foundry evaluators** | `eval.yml` → `foundry-evaluators` | Each evaluator ≥3.0/5 | Groundedness, relevance, coherence, retrieval (optional — requires `AZURE_OPENAI_ENDPOINT`) |
| **Red team** | `redteam.yml` (manual + weekly) | 0 high/critical, ≤10% overall | 8 adversarial families: prompt injection, jailbreak, off-domain, PII probe, citation skip, hallucination, token bomb |

Thresholds are pinned in `evals/calibration.json`. Do not tweak them to make CI green — follow the recalibration protocol in `evals/calibration.md`. Re-calibration goes through `.squad/decisions.md`.

## Development Workflow

**Local loop (Python services):**
```
pip install -e ".[dev]"
ruff check .
mypy --strict .
pytest -q
```

**Frontend:**
```
npm ci && npm run lint && npm run typecheck && npm test -- --run && npm run build
```

**Infra validation:**
```
az bicep build --file infra/main.bicep --stdout > /dev/null
```

**Eval cadence:** Run hermetic (cassette) gates locally before pushing. Use `EVAL_MODE=live` + deployed URL for integration smoke tests against a real subscription. When adding a scenario, add the matching cassette first.

**Deploy:** `azd auth login && azd up` (~15–20 min first run). CI uses `.github/workflows/deploy.yml` via OIDC on push to `main`.

## Governance

This constitution captures the non-negotiable contracts of the MTA AI Hackathon accelerator. It is derived from the repo's existing `README.md`, `docs/architecture.md`, `docs/voice.md`, `docs/evals.md`, and `.github/copilot-instructions.md` — not invented. Amendments require a decision entry in `.squad/decisions.md` and must update the enforcing files listed under each principle.

All PRs and code reviews must verify compliance with these principles. The CI gates (Quality Gates table above) are the automated contract — if a gate fails, the principle is violated.
